A Beginners Basic Guide to Making Content for Aurora Builder

Important Information: To start making content for Aurora it is highly recommended to use a 'text-editor program. There are a few to choose from:

  • VS Code: https://code.visualstudio.com/ (*Recommended)
  • Additionally install the Aurora Snippets extension for VS Code: https://marketplace.visualstudio.com/items?itemName=swdriessen.aurora-snippets
  • Notepad++: https://notepad-plus-plus.org/downloads/
  • OxygenXML: https://www.oxygenxml.com/
  • Atom: https://atom.io/
  • There is a Atom Package for Aurora Snippets by mrgskiier: https://atom.io/packages/aurora-snippets

Step 1: The File

Make a copy of an already already existing file that is close to the content you’re making into your custom/user folder.

Ex: I want to make a custom background so I would make a copy of background-criminal.xml from custom/core/player-handbook/ and place it in custom/user and now open the copied file (the file in custom/user) in the code editing program.

Step 2: Editing

Simply edit the open file to match the content that you need to add. To make more sense of what is going on a quick rundown of what everything is.

Every file has to have the xml header to be read properly by Aurora:

<?xml version="1.0" encoding="utf-8" ?>

Add an elements block (Yes there is a difference between an element block and an elements block, the element block is for a single feature while the elements block contains all the elements in the file, if you’re still confused between the difference it is element vs element(s)), this tells Aurora that everything within the element is to be added:

<?xml version="1.0" encoding="utf-8" ?>
<elements>
</elements>

*Remember for <elements> there should be a matching </elements> and for every <element> there should be a matching </elements>

Now to add an info block:

*This step is optional but if you want to host this file on any repositories the info block is required

<?xml version="1.0" encoding="utf-8" ?>
<elements>
   <info>
       <update version="0.0.1">
           <file name="background-criminal.xml" url="https://raw.githubusercontent.com/aurorabuilder/elements/master/core/players-handbook/backgrounds/background-criminal.xml" />
       </update>
   </info>
</elements>

Some of the first files would contain a description, name and author block, these are now obsolete and you can simply delete it if you’re copy-pasting from a base file.

Update number is used to update files since Aurora checks the local file against the remote file and if the remote file number is higher it will download the newer version. Thus if any changes are made to a file to update it you should increment the number one higher.

File name should match the file name, the url can be left blank if you don’t plan on adding it to any repositories, like so:

<file name="background-criminal.xml" url="" />

Again make sure that each <xxx> has a matching </xxx> unless the block has a <xxx /> in which case it doesn't need a matching closing block.

Now to start on the actual element. Add a new element line and add the following parameters:

  • Type: Spell, Background, Class, Class Feature, Archetype, Archetype Feature, etc.
  • Source: Self Explanatory
  • ID: this is unique to every element, if any elements have the same ID anything in the custom/user folder will overwrite what’s in the other folders. The basic formula for an id is ID_AUTHOR_SOURCE_TYPE_NAME so for example the Criminal Background from the Player Handbook would have the ID: ID_WOTC_PHB_BACKGROUND_CRIMINAL

(*To future proof against anything that will be added into the repo make sure the ID is unique enough that it will not ever be overwritten)

<?xml version="1.0" encoding="utf-8" ?>
<elements>
   <info>
       <name>Criminal</name>
       <description>Criminal Background</description>
       <author url="http://dnd.wizards.com/products/tabletop-games/rpg-products/rpg_playershandbook">Wizards of the Coast</author>
       <update version="0.0.6">
           <file name="background-criminal.xml" url="https://raw.githubusercontent.com/aurorabuilder/elements/master/core/players-handbook/backgrounds/background-criminal.xml" />
       </update>
   </info>


   <element name="Criminal" type="Background" source="Player's Handbook" id="ID_WOTC_PHB_BACKGROUND_CRIMINAL">
   </elements>
</elements>

Now within the element you have just created you can start adding parameters.

Anything within a <description></description> will dictate what will be shown in Aurora. This can be stylized using parameters such as <i> </i>, <b> </b>, etc.

<?xml version="1.0" encoding="utf-8" ?>
<elements>
   <info>
       <name>Criminal</name>
       <description>Criminal Background</description>
       <author url="http://dnd.wizards.com/products/tabletop-games/rpg-products/rpg_playershandbook">Wizards of the Coast</author>
       <update version="0.0.6">
           <file name="background-criminal.xml" url="https://raw.githubusercontent.com/aurorabuilder/elements/master/core/players-handbook/backgrounds/background-criminal.xml" />
       </update>
   </info>

   <element name="Criminal" type="Background" source="Player's Handbook" id="ID_BACKGROUND_CRIMINAL">
       <description>
       This is a description of a criminal.
       </description>
   </element>
</elements>

Some elements require setters, such as hd for Hit Dice if you’re making a class, etc.

Now to add a sheet block, this will dictate what shows up on the actual generated character sheet and will only show up on the character sheet. If it doesn’t require being shown on the sheet then you can simply add the parameter display=”false” and it will not print into the sheet. This block can also be edited the same way as the description markup to reflect onto the sheet.

<?xml version="1.0" encoding="utf-8" ?>
<elements>
   <info>
       <name>Criminal</name>
       <description>Criminal Background</description>
       <author url="http://dnd.wizards.com/products/tabletop-games/rpg-products/rpg_playershandbook">Wizards of the Coast</author>
       <update version="0.0.6">
           <file name="background-criminal.xml" url="https://raw.githubusercontent.com/aurorabuilder/elements/master/core/players-handbook/backgrounds/background-criminal.xml" />
       </update>
   </info>

   <element name="Criminal" type="Background" source="Player's Handbook" id="ID_BACKGROUND_CRIMINAL">
       <description>
       This is a description of a criminal.
       </description>
       <sheet display="false">
       </sheet>
   </element>
</elements>

This would make it so that it doesn’t show up in your sheet but if you need it to show up you can do it like so:

    <element name="Reckless Attack" type="Class Feature" source="Player's Handbook" id="ID_WOTC_PHB_CLASS_FEATURE_BARBARIAN_RECKLESS_ATTACK">
       <description>
           <p>Starting at 2nd level, you can throw aside all concern for defense to attack with fierce desperation. When you make your first attack on your turn, you can decide to attack recklessly. Doing so gives you advantage on melee weapon attack rolls using Strength during this turn, but attack rolls against you have advantage until your next turn.</p>
       </description>
       <sheet>
           <description>When you make your first attack on your turn, you can decide to attack recklessly. Doing so gives you advantage on melee weapon attack rolls using Strength during this turn, but attack rolls against you have advantage until your next turn.</description>
       </sheet>
   </element>

*Note that this is not a full file, this is simply an element to use as an example, putting just this into a xml file will not work; you would need the xml header and the elements block.

Using the level parameter will allow for the description to be changed based on level. Additionally you can also set an action parameter to set it as a Action, Bonus action or Reaction. For example:

    <element name="Retaliation" type="Archetype Feature" source="Player's Handbook" id="ID_WOTC_PHB_ARCHETYPE_FEATURE_PATH_OF_THE_BERSERKER_RETALIATION">
       <description>
           <p>Starting at 14th level, when you take damage from a creature that is within 5 feet of you, you can use your reaction to make a melee weapon attack against that creature.</p>
       </description>
       <sheet action="Reaction">
           <description>When you take damage from a creature that is within 5 feet of you, you can use your reaction to make a melee weapon attack against that creature.</description>
       </sheet>
   </element>

Adding a rules block will allow for the sheet to update as you level up on change based on variable stats, (DEX times per long rest, CON times per short rest, etc), this can be to insert information into the description and auto update, for example the rules block in the barbarian rage:

    <element name="Rage" type="Class Feature" source="Player's Handbook" id="ID_WOTC_PHB_CLASS_FEATURE_BARBARIAN_RAGE">
       <description>
           <p>In battle, you fight with primal ferocity. On your turn, you can enter a rage as a bonus action. While raging, you gain the following benefits if you aren't wearing heavy armor:</p>
           <ul>
               <li>You have advantage on Strength checks and Strength saving throws.</li>
               <li>When you make a melee weapon attack using Strength, you gain a bonus to the damage roll that increases as you gain levels as a barbarian, as shown in the Rage Damage column of the Barbarian table.</li>
               <li>You have resistance to bludgeoning, piercing, and slashing damage.</li>
           </ul>
           <p>If you are able to cast spells, you can't cast them or concentrate on them while raging.</p>
           <p class="indent">Your rage lasts for 1 minute. It ends early if you are knocked unconscious or if your turn ends and you haven't attacked a hostile creature since your last turn or taken damage since then. You can also end your rage on your turn as a bonus action.</p>
           <p class="indent">Once you have raged the number of times shown for your barbarian level in the Rages column of the Barbarian table, you must finish a long rest before you can rage again.</p>
       </description>
       <sheet action="Bonus Action">
           <description usage="{{barbarian rage:count}}/Long Rest">Advantage on Strength checks and Strength Saves.
           A +{{barbarian rage:damage}} to damage rolls with strength melee weapon attacks.
           You have resistance to bludgeoning, piercing, and slashing damage. </description>
           <description usage="Unlimited" level="20">Advantage on Strength checks and Strength Saves.
           A +{{barbarian rage:damage}} to damage rolls with strength melee weapon attacks.
           You have resistance to bludgeoning, piercing, and slashing damage.</description>
       </sheet>
       <rules>
           <stat name="barbarian rage:count" value="2" level="1" />
           <stat name="barbarian rage:count" value="1" level="3" />
           <stat name="barbarian rage:count" value="1" level="6" />
           <stat name="barbarian rage:count" value="1" level="12" />
           <stat name="barbarian rage:count" value="1" level="17" />
           <stat name="barbarian rage:count" value="93" level="20" />
           <stat name="barbarian rage:damage" value="2" level="1" />
           <stat name="barbarian rage:damage" value="1" level="9" />
           <stat name="barbarian rage:damage" value="1" level="16" />
       </rules>
   </element>

The rules block can also be used to grant abilities, allow for selections, edit statistical scores (Speed, AC, Ability score, etc) all of that information can be found at https://aurorabuilder.com/documentation/references/
